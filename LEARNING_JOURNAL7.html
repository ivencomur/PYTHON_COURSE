2) Key Answers from My Research (PDF)

2.1 Why Django Is Popular

Django enables fast, secure development without reinventing the wheel. It includes auth, admin, ORM, and templating out of the box, is written in approachable Python, and has excellent documentation and community support.

2.2 Five Companies Using Django (What for)

  • Instagram — user accounts, media uploads, feed management.
  • Spotify — internal dashboards and operational tooling.
  • NASA — public-facing content sites with strong security.
  • Disqus — global comment system serving millions.
  • Pinterest — content management for pins/boards and delivery.

2.3 When to Use (or Not Use) Django — Scenario Decisions

  • Multiple users? Yes — built-in auth and permissions.
  • Fast iteration? Yes — batteries-included, structured tooling.
  • Tiny app, no DB/files? No — use a lighter microframework (e.g., Flask).
  • Total, low-level control? Not ideal — Django is opinionated.
  • Big project, need support? Yes — great docs and community.

3) Environment Setup Completed

I verified Python, created/activated a virtual environment named achievement2-practice, and installed Django, confirming the version via django-admin --version.

Windows (Git Bash)

py -3.9 --version py -3.9 -m venv achievement2-practice source achievement2-practice/Scripts/activate python -m pip install --upgrade pip pip install "Django>=5" django-admin --version

macOS/Linux

python3 --version python3 -m venv achievement2-practice source achievement2-practice/bin/activate python -m pip install --upgrade pip pip install "Django>=5" django-admin --version

4) Problems I Ran Into & Solutions

  • PowerShell vs Git Bash: On Windows, I stuck to Git Bash for consistent activation syntax.
  • Multiple Python versions: Explicitly calling py -3.9 ensured venv creation with the expected interpreter.
  • Package updates: Running python -m pip install --upgrade pip prevented install warnings.

5) New Skills Acquired

6) What I'll Remember Most
