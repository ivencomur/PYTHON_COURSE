What I Learned from the Research

Django is a Framework

A framework is like a pre-built house structure. Instead of building every wall from scratch, you get the basic structure and just add your custom features. Django provides the structure for web applications.

MVT Architecture

Django uses something called MVT (Model-View-Template) architecture. This is just a fancy way of saying it separates your code into three parts:

  • Model – How data is stored (like recipes in a database)
  • View – The logic (what happens when someone clicks a button)
  • Template – What users see (the HTML pages)

Keeping these separate makes code easier to change later.

When NOT to Use Django

The most important thing I learned: Django isn't always the answer. For really simple programs that don't need a database or user accounts, Django is overkill. It's like using a semi-truck to go to the grocery store.

Virtual Environment

A virtual environment is like a separate workspace for Django. It keeps Django's files separate from other Python projects so they don't interfere with each other. Think of it like having a separate toolbox just for Django work.

Version Differences

The course shows Django 3.2.4 from 2021. I installed Django 5.x (current version from 2024). Using the newer version is actually better because:

  • It has the latest security fixes
  • Better error messages that help me learn
  • Skills I learn apply to current job market

I documented this difference in my screenshots to show I understand the versions don't match the course material.

Challenges I Had

Understanding the Terminology

Django uses confusing names. What Django calls a "View" is actually the logic part (other frameworks call this the "Controller"). What Django calls a "Template" is what others call a "View." Once I stopped worrying about the names and focused on what each part DOES, it made more sense.

Virtual Environment Commands

The course shows commands for "virtualenvwrapper" which doesn't work in Git Bash. I used the built-in "venv" instead, which works fine but has different commands. This is a common situation in programming—finding alternative tools when the suggested one doesn't work on your system.

Knowing When Django is Overkill

It was tempting to think "Django is great, so use it for everything!" But Question 3 taught me to think critically. Sometimes simpler tools are better. This reminds me of the saying: "If all you have is a hammer, everything looks like a nail."
